## Defects found & fixed

### DEFECT-01 (P0, isolation law) — path traversal escapes the tenant volume
**Found by:** Layer 1 `test_tenant_repo_dir_never_escapes_tenant_volume`, confirmed
empirically before the fix:

| input | old result | verdict |
|-------|-----------|---------|
| `tenant_id="/etc", repo="x"` | `/etc/repos/x` | escaped the volume entirely |
| `tenant_id="tenantA", repo="../../../tenantB/repos/secret"` | resolves into **tenantB** | **cross-tenant read** |
| `tenant_id=".."` | `…/../repos/x` | escaped tenant home |
| `repo_url` ending `/.` or with `\x00` | `repo_name_from_url` returned `.` / `\x00` | unusable / unsafe |

**Root cause:** `tenant_repo_dir` did `volume_root() / tenant_id / "repos" / repo_name`
with **no validation** — an absolute or `..`-bearing component reparents the path
(`Path("/root") / "/etc"` → `/etc`; `… / ".." / …` climbs out). This is precisely the
"no runtime cross-tenant read barrier" the sealed **AC-M2-001** finding names, and it
violates the CLAUDE.md isolation law ("one tenant never sharing with another").

**Fix (real code, `services/code_intel/src/code_intel/paths.py`):**
- Added `_safe_component()` — rejects empty, `.`/`..`, path separators, null bytes and
  absolute paths, raising `UnsafePathComponent(ValueError)`; applied to *both*
  identifiers in `tenant_repo_dir`.
- Hardened `repo_name_from_url` to sanitise to a single safe component (traversal
  specials collapse to `repo`).

**Evidence of fix:** Layer 1 goes 2-failed→**5-passed** (1000-example `thorough`
profile); ruff + `mypy --strict` clean; the sealed doc01 tests that touch clone/paths
still pass; the Layer 4 containment result corroborates it end-to-end. Commit `a8f2af5`.
