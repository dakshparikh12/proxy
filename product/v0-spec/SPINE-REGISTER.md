## (superseded) earlier template research (kept for reference)
Two files: (a) a lean project **CLAUDE.md** constitution (built once in Doc 00 — broad-only, "would removing this cause a mistake?", convert stable behaviors to hooks); (b) a self-contained **per-doc SPEC** in 8 sections:
1. Title + summarized ToC · 2. Context & intent (+ **Non-goals/do-not-build** boundary — this REPLACES the deleted "V0 scope/deferrals"; doc contains only in-scope) · 3. Named tools & stack (prefer CLI, absolute paths) · 4. Mechanism/design (named-tool + interface level, pattern files to imitate, invariants, NO code) · 5. **Acceptance criteria (CENTERPIECE)** · 6. Worked golden example(s) · 7. Real-data verification harness · 8. End-to-end done.
**AC method:** EARS backbone (WHEN/WHILE/WHERE/IF-THEN + SHALL/SHOULD/MAY + named threshold), one req/line, split compounds, ban vague adjectives, tag `[O]`utput(runnable check) vs `[P]`rocess(code review), invariant/property criteria for edge cases; 30-50 OK when each maps to one runnable check (count is an output not a target). AC drive an ATDD loop: implement→run all checks→iterate until green; un-green = defect.
**Verification:** real named repos (not stubs — SWE-bench Verified was retracted); ABC-Bench two-stage guard (every test passes-on-correct AND fails-on-broken); stated pass^k target; read failure transcripts not just pass rates; baseline-then-iterate.
**Anti-patterns:** over-stuffed spec, inlined code, vague adjectives, compound "and" criteria, examples-as-whole-test-set, self-grading on synthetic fixtures, tests-that-can't-fail, trusting aggregate pass rates, "looks done" verification. Dossier: research/dossier/synthesis-agentic-build-spec-template.md. Doc 01 = the worked exemplar.

## DEFERRED DESIGN — Proactive (V1; finalized 2026-07-16, do not re-derive)
Structured judge+gate, NOT one prompt (evidence: LLMs fail zero-shot multi-party interjection — must be structured). Pipeline: Doc 03 material-change events → **the judge** (cheap model, fires on material events only, ~90% NOTHING, default silent): "Would a sharp, respected participant speak here? What, and in what general direction?" — infers setting from the enriched notes (firmness/reversibility/contradiction), NEVER meeting-type labels. Contribution lenses as inspiration not menu: validate·correct·quantify·gap/risk·ideate(incl. design-elevation)·unblock·encourage. Develops via the SAME workroom (read/verify-tier, bounded real work allowed); never speaks unverified. **Deterministic gate owns the mouth**: still-relevant-now · not-already-said · clears dial · natural boundary; late→banked for close, never stale. Hard floors bypass judge (decision-final-on-checkable, contradiction, commitment-on-wrong-number) → surface with proof. Whisper-first (person gets first move via DM where supported). Dial V0-simple: off/normal/lead; learned per-room dial later (needs ≥30-meeting corpus). Excluded: process-policing, meeting-type modes, unverified speech, mid-sentence interruption. Cost: cents/hr.
**Hooks that STAY in V0 so proactive drops in cleanly:** Doc 03's material-change event emitter (built, cheap, consumer deferred) · Doc 04's delivery priority rules (written generically: human-answer > hard-floor > gated) · the workroom's read-tier entry (Doc 05 §2.6) · the tile's "has-something" signal (Doc 02).

## Locked decisions log
*(filled as docs lock; seed = the grounded core stack, 2026-07-15)*
- Architecture: one persistent thin **Orchestrator (Proxy)** + ONE universal **Workroom** (everything flows through) + **Index** + enriched **Notes** + **Voice**. Everything routes through Proxy.
- Index (LOCKED v5 — SIMPLE global solution + on-demand LSP): "Claude Code on the customer's repo." THREE pieces = (1) blobless clone on our infra, (2) lightweight tree-sitter orientation map (where areas/symbols are + CODEOWNERS ownership; the map, not the brain), (3) the Claude-Code-style agent doing find/answer/work LIVE = grep+read+reason + **the LANGUAGE SERVER on demand** (go-to-def/find-references/call-hierarchy = blast-radius, triggered when needed; adopt Serena/solid-lsp, warmed on sandbox boot; the LSP IS the reference index — we don't build one) + sandbox. NO pre-built reference graph, NO SCIP/hydration/Zoekt in V0. "What breaks" = WITHIN-repo via LSP find-references, returned as an HONEST LOWER BOUND (flag: into-third-party-libs needs deps [optional, agent can install]; dynamic/config/cross-service may be missed). Trust = "never overstate the evidence." Verify = SMOKE TEST. FULL end-goal core incl. blast-radius.
- Access (LOCKED v4): GitHub App (contents:read) — Nango = credentials + future connectors (NOT the git path; RepoProvider iface owns clone/webhook). Blobless clone (--filter=blob:none, blame works). Fresh = webhook + reconcile-at-meeting-start. Per-meeting pinned to ONE base SHA (or PR head); mid-meeting push → notify, don't mutate. Secrets excluded from index/context/sandbox + redact (bytes may remain in object store). Only the disposable sandbox runs repo code.
- ANY-SCALE (LOCKED principle): the design works on a 10k-LOC repo AND a 10M-LOC monorepo. Scale changes TIME, not CAPABILITY — a hard blast-radius on a huge repo may run async ("I'll trace that"), like Claude Code. Agent always works on the RELEVANT SLICE, never loads the whole repo.
- SPEED TIERS (same capability, prepared AHEAD on connect/push, scaled by REPO SIZE not customer tier — NOT deferred capabilities): every repo = clone+map+pre-warm-LSP; large repo also gets precomputed SCIP + Zoekt (built ahead, off meeting critical path) so precise queries are instant instead of after a wait; hydration (dep install) where cross-library precision needed. Without the precompute the agent still answers, just slower. We NEVER say "repo too big."
- GENUINELY LATER (not index scale): cross-REPO linking (multi-repo meetings); compiler-oracle/mutation verification; full compliance (but start SOC2 T2 observation window early).
- Workroom: one Claude Agent SDK loop, zero-LLM gate → effort-routed reasoning → E2B sandbox; prompt-cache prefix; staged-draft actions; verify cascade (deterministic→HHEM→fresh-context judge).
- Orchestration: asyncio Task-per-workroom, no message bus, semaphore 3-5; delivery headline→voice / detail→chat / artifact→screen.
- Notes: two-speed Haiku live + Sonnet close; enriched, observed-vs-inferred; referents→code.
- Voice: Recall + AssemblyAI passthrough (BYOK $0.15) + Cartesia TTS; skip Pipecat/LiveKit.
- Proactive: structured VOI gate (not one prompt) + Silero VAD/Smart Turn v3 + whisper-first; V0 dial off/semi/lead.
- Cost: ~$0.95–1.15/hr, STT-gated; enforced live by `check_meeting_budget()` (persisted `meeting_cost`, soft-degrade + hard-cap-to-notes-only).
- **Platform adoption (2026-07-16):** adopt the `~/platform` sibling repo's production substrate broadly (GCP infra, SDK-isolation triad, broker-free durable-ops, session durability, transport/security spine, MCP tooling, GCS versioning) — engineering substrate, not the product.
