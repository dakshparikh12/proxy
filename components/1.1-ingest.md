# Proxy · Component 1.1 — Ingest
### One doc = the spec. Read `AGENTS.md` first. Built via the loop in `docs/BUILD-SYSTEM.md`. Fixtures in `/fixtures/1.1/`, eval in `/eval/1.1/`.
### Every acceptance criterion is arbiter-tagged: [pytest] = inner-loop code arbiter · [eval] = section real-data arbiter.

## 1. Goal
Turn any connected source — code, docs, tickets, wiki, infra, or a dragged-in file — into a uniform stream of **cited, permission-stamped, normalized Artifacts**, with all explicit structure and cross-references preserved, so the graph can be built without ever re-reading the source. One law governs it: **lossless-or-honest** — extract faithfully with a citation, or emit a typed `Unknown`; never a silent drop, never a fabrication, never a silently-wrong Artifact.

## 2. Product (context + end goal)
**What it is.** Proxy's universal front door. A customer connects a source (GitHub, Jira, Notion/Confluence, Drive, Slack, infra files) or drags in a file; 1.1 produces clean, cited, permission-tagged, normalized knowledge — while never keeping a copy of their estate, never leaking a secret, and honestly flagging what it can't parse. It is the *ingest and normalize* stage: it makes every source uniform and extraction-ready. It does **not** interpret meaning — semantic entity resolution belongs to 1.2 (see §3).

**Why it matters downstream.** 1.1 is the first mile of the whole system: every later layer inherits its quality ceiling. An object 1.1 silently drops is a question Proxy can never answer. A mis-stamped permission is a breach. A corrupted extraction becomes a confidently-wrong meeting claim three components later. A cross-reference 1.1 flattens into prose (the wiki page that *linked* to a Jira ticket) is an edge 1.2 can never recover — because the raw source is gone (zero-copy). Its outputs — the **Artifact stream**, the **permission graph**, and the **honest Unknowns ledger** — are the only raw material 1.2 ever sees.

**Features.**
- Connect any common source read-only in an afternoon (not a 3-month review); a source with no pre-built connector is handled by an explicit switch (§5), not silently promised.
- Pull two streams from every source: **content** and the **permission graph** (who can see what).
- Redact secrets before anything is stored.
- Extract faithfully via **two lanes**: a deterministic code lane (fast, exact, cheap) and a single agentic reader lane for everything else (self-correcting on hard inputs); anything unparseable becomes an honest `Unknown`.
- Preserve **explicit structure** (code call/import edges) and **raw cross-references** (links, `see X`, @mentions) as stamped Artifact attributes — the material 1.2 resolves into edges.
- Stamp every piece with a citation to its exact origin, its source ACL, a tenant tag, provenance, and a content-hash id.
- Store only derived understanding (zero-copy); a regulated customer can run a mode that keeps nothing raw.

**Qualitative flow.** Customer connects → we hold the key in self-hosted Nango → we crawl content + permissions (initial full, then deltas, deletes reconciled) → we redact secrets → we route each object to a lane: **code** goes to deterministic tree-sitter extraction; **everything else** goes to one agentic reader that reads it cheaply, self-checks faithfulness, escalates hard inputs to vision only when needed, and emits a faithful Artifact or an honest `Unknown` → we stamp (citation, ACL, tenant, provenance, content-hash) → we emit derived-only Artifacts to 1.2. Nothing raw persists; truth is re-read live later. Permissions are enforced at read time, so you only ever see what you already could.

## 3. Not-Included (scope boundary)
- **No semantic understanding / entity resolution** (that's **1.2**). 1.1 emits *explicit* structure (parser-certain code edges) and *raw* references (the literal link/mention with its citation) — it never decides that "checkout" in a ticket resolves to node #4471, or that a link *means* `depends_on`. Interpretation requires the whole graph in view, which 1.1 (local, per-source, parallel) does not have.
- No graph/nodes unification, community detection, clustering, hierarchy, or naming (1.2 / 1.3).
- No indexing/embeddings, no answer generation, no sandbox task execution (1.2 / 05).
- No writing to or mutating any source (read-only, always).
- No per-vertical or per-size logic.

## 4. Tech stack (this component)
Inherits `AGENTS.md §Stack`. Component specifics:
- **Access:** **Nango** (self-hosted; the only credential holder — every read passes through it; it is the auth/token layer, *not* the crawler).
- **Crawl / sync (behind the `Source` seam):** **git-native** (`git fetch` + SHA-diff) for the code lane — deltas, deletes, and crash-resume come free from git; **dlt** (open-source Python, incremental + merge/delete + resumable state) for non-code families (tickets, wiki, Drive, Slack, infra APIs). One seam, two+ adapters.
- **Code parse:** **tree-sitter** (incremental, ~40+ langs) for declarations/calls; **SCIP** for exact `file:line`-cited cross-references. Behind the `GraphExtractor` seam so either is swappable. Content-hash (e.g. XXH3) for incremental re-index.
- **Everything-else parse (one agentic reader loop):** **Claude Agent SDK** loop with a small read/emit tool belt: `docling_parse` (**Docling**, deterministic doc fast-path), `vision_read` (self-hostable VLM behind the metered gateway, escalation-only), `run_in_sandbox` (**E2B**, config/infra probing), `emit_artifact`, `note_unknown`. **Apache Tika** as the `LONGTAIL_FORMAT` fallback.
- **Redact:** offline secret scanner (**Betterleaks/Gitleaks-class**, regex+entropy, no network calls), pre-persist hard gate.
- **Store:** Postgres 16 + pgvector behind the **Store adapter** (the only DB access).
- **Model calls:** all through the metered gateway `src/proxy/llm.py`. Batch API for the reader lane where latency allows.
Seams: `Source`, `GraphExtractor` (§6).

## 5. Architecture & stages (direction-pointers on the HOW; precise on the contracts in §6)
```
connect (self-host Nango)
  → crawl (behind Source seam): git-native (code) ‖ dlt (non-code) — initial + incremental + deletes + idempotent + crash-resume
  → scope (respect ignore/vendor/generated markers; exclude non-owned code — deterministic)
  → redact (offline secret scan, BEFORE persist — deterministic hard gate)
  → route by object type:
        CODE  → tree-sitter + SCIP  → explicit entities + EXTRACTED edges, cited        (deterministic lane)
        ELSE  → ONE agentic reader loop {docling_parse → self-check → vision_read on low-conf → emit | note_unknown}
                 preserves raw references (links/mentions) as attributes             (agentic lane, batch-parallel)
  → stamp (content-hash id FROM SOURCE BYTES + citation + acl_ref + tenant + provenance)
  → emit (derived-only, zero-copy) → 1.2
```
**Per-stage non-negotiables:**
- **connect** — deterministic; Nango holds every credential; no key ever reaches an Artifact, log, or the sandbox.
- **crawl** — the `Source` seam yields content **and** permission items; never throws on empty; permission sync runs on its **own cadence**, decoupled from content re-parse.
- **scope** — deterministic ignore layer; vendored/generated/third-party code is excluded so the graph isn't flooded with noise the company doesn't own.
- **redact** — runs **before** persist, always; deterministic only (an LLM on the secret decision is a vulnerability).
- **code lane** — deterministic and exact; cheaper and more reliable than any agent; code never enters the agentic lane.
- **agentic reader lane** — **cheap deterministic read FIRST**; the model (vision) fires **only** on genuine low-confidence; the agent's tool belt carries **zero authority-bearing tools** (no ACL/permission/tenant mutation) — injection can at worst yield a garbage Artifact caught by the faithfulness self-check, never a permission change; **faithful-or-Unknown** is mandatory (verify extraction against the source span; mismatch → `Unknown`, never a guess).
- **stamp** — the content-hash id is computed from the **source bytes**, not the agent's output, so the non-deterministic lane is still **idempotent at the identity level** (same source in → same id out).
- **emit** — derived-only; no raw source file persists beyond a transient warm `raw_ref`.

**Escalation switches (OFF by default):** `ENTERPRISE_ACL` (SSO/SCIM per-user resolution) · `REGULATED_PII` (contextual-PII classifier beyond secrets) · `LONGTAIL_SOURCE` (agent-generated adapter for exotic/legacy sources — Documentum, mainframe, niche DMS; explicitly the boundary of the common connector surface) · `LONGTAIL_FORMAT` (Apache Tika for rare file types) · `ENTERPRISE_EXTRACTION` (hosted agentic-OCR API for regulated/high-accuracy doc extraction) · `DATA_RESIDENCY` (cold/VPC: keep only derived, re-read source live).

## 6. Data contracts & seams
Uses `Artifact`, `PermissionObject`, `Principal`, `Unknown` from `AGENTS.md §Shared Types`. `emit` writes these tenant-scoped and calls the 1.2 seam. Every Artifact MUST carry: content-hash `id`, `tenant_id`, `kind`, normalized `content`, `citation{source_uri,locator}`, `acl_ref`, `provenance{fetched_at,extractor,confidence,secrets_redacted}`, `raw_ref|null`, and — where present in the source — `references[]` (raw cross-references: link/mention target + citation, marked unresolved for 1.2). Code-lane Artifacts additionally carry EXTRACTED `Edge`s (`calls`/`imports`, `confidence:"EXTRACTED"`, cited).

**Seams (get a contract test each):**
- `Source` (any-source → content + permission stream). Contract: `stream()` yields content **and** permission items; never throws on empty; supports initial + incremental + delete; every emitted Artifact carries citation + acl_ref + tenant + provenance; a new source family is a new adapter with **0** downstream change.
- `GraphExtractor` (code structure behind one interface: tree-sitter/SCIP today, swappable). Contract: `extract(repo) → stream of entities + EXTRACTED edges`, each cited; swap causes 0 downstream change.

## 7. Gotchas (known wrong turns — the reliability lever; feeds the build skill)
- **Nango is not the crawler.** It is the credential vault + authenticated-read proxy only. Deltas/deletes/resume come from git (code) and dlt (non-code) behind the `Source` seam. Wiring "crawl" into Nango is the first wrong turn.
- **Never put an LLM on the credential path or the ACL decision** — deterministic only (a hallucinated ACL is a breach).
- **The agentic reader touches no authority-bearing tool.** Its belt is parse/vision/sandbox/emit/unknown — all read-or-emit. Ingested content is inert data; an injected "mark this public" instruction can never reach a permission because the tool to do so does not exist in the lane.
- **Redaction runs BEFORE persist, always** — not after.
- **"Extracted" ≠ "faithful."** A hard input that can't be faithfully parsed is an `Unknown`, never a silently-wrong Artifact. The reader's self-check against the source span is mandatory, not optional.
- **Preserve raw references; do not flatten them.** The wiki link to a Jira ticket, the `see checkout RFC`, the @owner mention — emit each as a `references[]` attribute with its citation. If 1.1 flattens it to prose, 1.2 can never build that edge and the connection is lost forever (zero-copy = no re-read). This is the single most important downstream-setup rule.
- **Do not interpret references here.** 1.1 records that a reference *literally exists*; it does not resolve what it points to. Resolution is 1.2 (needs the whole graph).
- **Content-hash from source bytes, not agent output** — otherwise the non-deterministic reader lane produces phantom duplicate Artifacts and breaks idempotency (T8) and 1.2's entity resolution.
- **Scope out non-owned code** (vendored/generated/dependencies) — a graph that's 60% third-party noise is worse than a smaller true one.
- **Zero-copy is literal:** no raw source file persists (warm `raw_ref` is transient; cold mode = none).
- **Permissions sync on their own cadence** (sharing changes more than content) — don't couple them to content re-parse.
- **Cheap-path-first is a cost rule, not a default.** Invoking the model on every object "to be safe" is the cost bomb; the reader must try the deterministic read and escalate only on low confidence.

## 8. Acceptance criteria (precise + general properties, verified on real data — never "exists")
*Read each as: "this must be true WHEN this condition holds." Qualitative and general — no implementation prescribed — but specific enough to be gradeable against a real estate. Claude Code chooses HOW; these define WHAT-must-hold.*

**Tier-1 — technical [pytest]** (deterministic graders on data built from a real estate; binary pass/fail)

*Safety & correctness of the atoms*
- T1. [pytest] WHEN a real estate is ingested, THEN **0** detected secrets appear in any Artifact, store, or log (grader: independent secret scan of all outputs).
- T2. [pytest] WHEN any Artifact carries a citation, THEN its locator resolves to source content that supports it — **0** unresolvable or non-supporting citations (grader: resolve every locator against pinned source).
- T3. [pytest] WHEN a cross-tenant query is issued, THEN it returns empty (not an existence-leaking error) and **0** cross-tenant reads succeed (grader: access matrix).
- T4. [pytest] WHEN ingest of an estate completes, THEN `count(Artifacts) + count(Unknowns) == count(in-scope source objects)` — no silent drop (grader: reconciliation vs direct source listing).
- T5. [pytest] WHEN warm mode runs, THEN **0** raw source files persist beyond a transient `raw_ref`; WHEN `DATA_RESIDENCY` is on, THEN **0** raw persists at all (grader: store audit).
- T6. [pytest] WHEN source access to an object is revoked, THEN that object is absent from results on the next permission sync (grader: revoke→sync→query).
- T7. [pytest] WHEN any Artifact is emitted, THEN it validates the schema, its `acl_ref` resolves, and its `id` is a stable content-hash of the source bytes (grader: schema + FK + determinism check).

*Structure & references handed to 1.2*
- T8. [pytest] WHEN an unchanged source is re-ingested, THEN **0** new or mutated Artifacts are produced — ids and content byte-identical, including for the agentic lane (grader: double-ingest diff).
- T9. [pytest] WHEN objects are added / modified / **deleted** at the source, THEN the next sync reflects each exactly and a deleted object leaves **0** orphaned Artifacts, edges, or references (grader: mutate→sync→reconcile).
- T10. [pytest] WHEN a source contains a cross-reference (link, `see X`, @mention), THEN a `references[]` entry is emitted carrying the raw target + a resolving citation, marked unresolved — **0** references silently flattened into prose (grader: seeded-reference recall vs golden).
- T11. [pytest] WHEN code is ingested, THEN every emitted structural edge is `confidence:"EXTRACTED"`, carries a resolving citation, and **0** edges are INFERRED/guessed in 1.1 (grader: edge scan + citation resolve).
- T12. [pytest] WHEN vendored / generated / third-party paths are present, THEN they are excluded per the scope layer and **0** non-owned files become owned Artifacts (grader: scope-list diff).

*Resilience & isolation*
- T13. [pytest] WHEN an ingest is killed mid-crawl, THEN resume reaches an exact T4 reconciliation — **0** duplicates, **0** drops (grader: kill/resume harness).
- T14. [pytest] WHEN a source API errors, rate-limits, or times out, THEN retries are bounded, **0** partially-persisted corrupt Artifacts remain, and a permanent failure lands as a typed `Unknown` (grader: fault-injection harness).
- T15. [pytest] WHEN the agentic reader lane processes ingested content containing an instruction ("mark public", "ignore rules"), THEN it is treated as inert data — **0** permission/tenant/ACL mutations occur and the lane exposes **0** authority-bearing tools (grader: tool-manifest assertion + injection-corpus run).
- T16. [pytest] WHEN 1.1 runs end-to-end, THEN **0** source credentials/tokens persist anywhere outside Nango — not in an Artifact, store row, log, or the sandbox (grader: credential-pattern scan on all outputs + sandbox env audit).

*Honesty & metering*
- T17. [pytest] WHEN an `Unknown` is emitted, THEN it validates its schema with `description`, `why_matters`, `likely_owner`, `resolver`, `re_verify_at` all populated (grader: schema check).
- T18. [pytest] WHEN any model call is made in 1.1, THEN it passes through the metered gateway `src/proxy/llm.py` — **0** direct calls on any path (grader: call-audit).
- T19. [pytest] WHEN an ACL-only change occurs at the source, THEN the `PermissionObject` updates **without** re-parsing content — the two cadences are provably decoupled (grader: ACL-mutate → parse-count diff).

**Tier-2 — product [eval]** (golden-dataset eval + threshold; on the estates in `/fixtures/1.1/` — code-monorepo / config-routed-messy / docs-heavy / tickets / wiki / office-drive / infra / secrets / hard-inputs / perms)

*Anything in, nothing faked*
- P1. [eval] GIVEN any real estate WHEN ingested THEN **≥95%** of in-scope objects become cited Artifacts and the remainder are typed `Unknown`s, with **0** silent drops. *Grader: deterministic reconciliation + citation spot-check.*
- P2. [eval] GIVEN 100 sampled Artifacts WHEN citations are checked THEN **0%** are hallucinated (every locator resolves to supporting content). *Grader: deterministic resolve + LLM-judge on 20.*
- P3. [eval] GIVEN the hard-input set (scanned PDFs, borderless tables, diagrams) WHEN parsed THEN each yields a faithful Artifact at **≥90%** field accuracy vs golden, OR an explicit `Unknown` — **never** a silently-wrong Artifact. *Grader: deterministic field-match vs golden key.*
- P4. [eval] GIVEN 50 sampled clean inputs across kinds (code, markdown, office docs, native PDFs) WHEN normalized THEN content is **≥98%** faithful to source — no truncation, reordering, or loss. *Grader: deterministic content-diff fidelity + LLM-judge on 10.*

*Every input family, one contract*
- P5. [eval] GIVEN each supported source family — git repo, issue tracker/tickets, wiki/pages, office/drive files, infra-as-code, and the manual drag-in door — WHEN ingested THEN each maps to the **same** Artifact contract at **≥95%** reconciliation per family, with **0** family-specific downstream branches. *Grader: per-family reconciliation + contract validation.*
- P6. [eval] GIVEN a source family with no pre-built connector WHEN `LONGTAIL_SOURCE` is OFF THEN 1.1 declines honestly with a named boundary (never a silent partial ingest); WHEN ON THEN it ingests to the same contract. *Grader: boundary-behavior assertion.*

*Structure & references — the 1.2 handoff*
- P7. [eval] GIVEN a code estate WHEN ingested THEN the emitted EXTRACTED call/import edges match a senior-engineer golden structural map at **≥90%**, all cited. *Grader: deterministic edge-set match vs golden.*
- P8. [eval] GIVEN a docs/wiki/tickets estate with real cross-references WHEN ingested THEN **≥95%** of human-identified references are preserved as `references[]` with resolving citations (unresolved is correct; *missing* is the failure). *Grader: reference-recall vs golden.*

*Permissions, secrets, freshness*
- P9. [eval] GIVEN two principals with different per-object access WHEN each queries THEN each retrieves **only** what the source permits, and revocation propagates on next sync. *Grader: deterministic access-matrix eval.*
- P10. [eval] GIVEN a repo seeded with 20 realistic secrets WHEN ingested THEN **100%** of the seeded set is redacted before persist, **0** leaked. *Grader: deterministic scan vs injection manifest.* (Threshold is against the **seeded** ground-truth set — the honest, gradeable bar; open-world recall is a monitored metric, not a gate.)
- P11. [eval] GIVEN a content change or deletion at the source WHEN the next sync runs THEN it is reflected end-to-end within **one sync cycle** on every estate. *Grader: mutate→sync→diff.*

*Same door, honesty, speed, cost*
- P12. [eval] GIVEN a 1-repo estate and a large multi-source estate WHEN both ingest THEN both complete through the **identical** code path — **0** size/industry branches. *Grader: code-path trace + config diff.*
- P13. [eval] GIVEN emitted `Unknown`s WHEN sampled THEN **100%** name a concrete reason and a plausible resolver/owner — **0** vague "couldn't parse." *Grader: schema + LLM-judge on sample.*
- P14. [eval] GIVEN first full ingest WHEN measured THEN estate-single ≤**15 min**, estate-monorepo ≤**2 h**; incremental delta p95 ≤**60 s** per changed object *(provisional — ratify at baseline per `docs/TESTING-STRATEGY.md §Gate`, then fixed)*. *Grader: Langfuse timing.*
- P15. [eval] GIVEN the agentic reader lane on a docs/tickets estate WHEN measured THEN **≥95%** of objects are served by the **deterministic** (non-model) read and the model fires only on the hard remainder, keeping cost per 1k objects under the per-estate-class ceiling recorded at baseline with **no regression** thereafter. *Grader: gateway meter + model-invocation-rate trace.*

**Done** = all Tier-1 pass + all Tier-2 clear threshold on every estate + adversarial suite clean (§10).

## 9. Real-data eval (the product arbiter; the part that proves it works)
**Golden dataset (`/fixtures/1.1/`).** Real, diverse, public/licensed estates pinned to exact commit SHAs (see `design/estates.md`), spanning the FULL supported input space — not just clean repos. The roster, each chosen to break a specific part of the system:

| Fixture role | Estate (pin SHA at seed) | Breaks / proves |
|---|---|---|
| `estate-single` | `psf/requests` | small clean baseline; goldens easy to author/dispute |
| `estate-monorepo` | `facebook/react` (`packages/*`) | scale + vendored/generated exclusion (T12), same-door at size (P12), speed (P14) |
| `estate-messy` | `home-assistant/core` (scoped subtree; 2000+ config-routed integrations via `discovery.async_load_platform`) | config-routing defeats static parse → guarantees non-empty `Unknown`s (P1/P13) and reference-preservation stress |
| `estate-docs` | `fastapi/fastapi` (huge `docs/`) | docs-heavy fidelity (P4) + doc↔code references (P8) |
| `estate-tickets` | small synthetic Jira/issue export (contract-parity fixture) | tickets family (P5) + ticket→code references (P8) |
| `estate-wiki` | small Confluence/Notion-style export (contract-parity fixture) | wiki family (P5) + macro/collapsed-table hard inputs |
| `estate-office` | small Word/Excel/PPT + native-PDF set | office/drag-in family (P5), clean-input fidelity (P4) |
| `estate-infra` | Terraform/k8s dir (from `estate-service`) | infra-as-code family (P5) |
| `estate-hard` | 3 scanned PDFs, 2 borderless tables, 2 diagrams (public/CC) | faithful-or-Unknown on hard inputs (P3) |
| `estate-secrets` | fork of `estate-single` + 20 injected secrets | redaction ground truth (P10) |
| `estate-perms` | synthetic 2-repo micro-estate + access matrix | ACLs/revocation (P9), cross-tenant (T3) |
| `estate-live` | fork of `estate-single`, mutated post-ingest | delta/delete/freshness (T9, P11) |

Author **~100 human-reviewed goldens** across P1–P15 (see `design/estates.md` workload table; add: structural-map goldens for P7, reference-recall goldens for P8/T10, injection corpus for T15). Living asset — every real failure gets added back.

**Grader independence (non-negotiable).** Graders recompute every fact from the pinned source of truth — reconciliation from a direct source listing, citations by independently resolving locators, secrets by scanning outputs, structural edges by an independent parse, references against a human-marked golden, timing from Langfuse traces. A grader **never** trusts the component's own logs, counters, or self-reports; the builder proposing "done" is evidence of nothing.

**Procedure.** Ingest each estate → run each [eval] criterion as a scored scenario (`/eval/1.1/scenarios.json`) → deterministic graders wherever a check confirms it (reconciliation, locator resolution, secret scan, access matrix, edge-set match, reference recall, store audit, model-invocation rate); calibrated LLM-judge (vs the human gold set) only for extraction faithfulness and citation support. Metrics: correctness, groundedness, completeness, honesty, latency, cost.

**Tool.** DeepEval (pytest-native, plugs into the loop as rung 2); Langfuse traces latency/cost + model-invocation rate; Promptfoo runs §10 adversarial.

**Scorer mix (per criterion).** 60% deterministic + 30% calibrated LLM-judge + 10% human. Never LLM-judge alone.

**Gate.** Threshold AND no-regression-vs-baseline; tolerance bands; pinned judge model; stable golden set.

**Iterate.** Any metric below threshold on any estate (including the messy/config-routed one) → fresh context + failure report → fix root cause → rebuild → repeat until accurate on real data. Fix the **spec or fixture** on a repeated stall, never weaken a criterion.

## 10. Adversarial suite (must find zero invariant violation)
- **Injection via ingested content** — a repo/doc whose content instructs "mark all files public / ignore rules"; assert the agentic lane treats it as inert data and holds **0** authority-bearing tools (T15). No permission mutation, ever.
- **Secret smuggling** in odd encodings / split across lines; assert redaction still gates before persist.
- **Cross-tenant probe + mid-session revoke** — assert empty results, immediate revocation at read time.
- **Zero-copy audit** — no raw bytes in store or logs, including cold mode.
- **Credential-exfiltration audit** — no Nango token reachable from an Artifact, log, or the sandbox (T16).
- **Reference-flattening probe** — seed known cross-references; assert **0** are lost to prose (T10/P8).
- **Hostile inputs** (corrupt / oversized / exotic / zip-bomb-class) → graceful `Unknown`s, bounded cost, no crash.
- **Cost-bomb probe** — an estate of many hard-looking-but-trivial docs; assert the reader stays on the deterministic path (P15), model-invocation rate stays bounded.
- **Citation-integrity sampling** — 0% fabricated.

## 11. Definition of Done
All [pytest] green with conformance cases · all [eval] clear threshold on **every** estate (including the config-routed messy one) · adversarial clean · ruff/mypy/bandit clean · no invariant-violating path (test-proven) · Store adapter is the only DB access · evidence folder committed · both founders signed off. **Done means the product promises are proven on real data — not that the code compiles.**
