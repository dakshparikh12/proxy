## Defects found & fixed

### DEFECT-01 (P0, isolation law) — path traversal escapes the tenant volume
Found by Layer 1, confirmed empirically before the fix:

| input | old result | verdict |
|-------|-----------|---------|
| `tenant_id="/etc"` | `/etc/repos/x` | escaped the volume entirely |
| `repo="../../../tenantB/repos/secret"` | resolves into **tenantB** | **cross-tenant read** |
| `repo_url` ending `/.` or with `\x00` | `repo_name_from_url` → `.` / `\x00` | unsafe |

**Root cause:** `tenant_repo_dir` joined identifiers with no validation, so an absolute
or `..`-bearing component reparented the path — the "no runtime cross-tenant read
barrier" the sealed **AC-M2-001** finding names; violates the isolation law.

**Fix (`services/code_intel/src/code_intel/paths.py`):** `_safe_component()` rejects
empty/`.`/`..`/separators/null/absolute (raising `UnsafePathComponent`), applied to both
identifiers; `repo_name_from_url` sanitised to a single safe component.

**Evidence:** Layer 1 2-fail→**5-pass** (`thorough` 1000 examples); ruff + `mypy
--strict` clean; sealed doc01 clone/path tests still pass; Layer 4 containment
corroborates it. Commit `a8f2af5`.
