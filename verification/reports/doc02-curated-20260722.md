## Defects found & fixed

### DEFECT-02 (robustness) — webhook parser crashes on malformed payload
Found by Layer 1, confirmed empirically:

| input | old result |
|-------|-----------|
| `{"data": None}` / `{"data": 123}` | `AttributeError: … has no attribute 'get'` |
| `{"data":{"participants":["notadict"]}}` | `AttributeError` |

**Root cause:** `meeting_metadata` (and the shared `data.get(...)` pattern in
`is_bot_removed`) assumed dicts; a corrupt/partial Recall delivery raised straight out of
ingest — the untested AC-FAIL webhook-robustness area the sealed verdict flagged.

**Fix (`services/transport/src/transport/events.py`):** `_as_dict()` coercion; the three
parsers degrade a non-object `data`/`participants`/entry to empty fields and coerce names
to `str`. Well-formed classification unchanged.

**Evidence:** Layer 1 1-fail→**6-pass** (`thorough` 1000 examples); ruff + `mypy
--strict` clean; **28** sealed doc02 event/meeting/webhook tests still pass. Commit `8862027`.
