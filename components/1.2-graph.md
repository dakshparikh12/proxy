# Proxy · Component 1.2 — Graph Construction (The Estate Graph)
### One document. Read `AGENTS.md` first. Consumes 1.1's output. Fixtures in `/fixtures/1.2/`.

---

## 1. Goal
Turn 1.1's Artifacts into a **hierarchical, entity-resolved, verified map** of the customer's whole system (code + docs + tickets + infra + people), and serve it so any downstream demand composes over it in <1s. **Not a store of truth** (the map holds the *address* of truth; live reads hold the value). **Not a fixed API** (six primitives compose; a new capability adds zero graph code).

## 2. Product (what it is, features, how it flows)
**What it is.** The context-and-capability substrate the whole meeting product runs on. A light, always-fresh **index** that lets an agent instantly find the right thing, understand it, check whether it's still true, and route a task to whatever can act on it. Store the map, query the territory.

**Features.**
- One unified map across code + docs + tickets + infra + people — the same real thing across sources is one node (entity resolution).
- Answers the relational questions similarity search can't: "what breaks if we change X?", "who owns this?", blast-radius, contradictions.
- Cited briefs — short plain-English understanding per unit the agent reads to get oriented fast.
- Live truth — asserts facts only from a live read of current source, cited; never a stale figure.
- Honest gaps — what it doesn't know is a named `unknown` with a likely owner.
- Six abilities everything composes over: **resolve · relate · explain · verify · operate · coverage**.

**Qualitative flow.** 1.1 Artifacts → extract structure (code) + knowledge (docs) → unify + entity-resolve into one graph, weave in permissions, build a file→unit→domain→system hierarchy → recover the edges parsers miss (or flag as `unknown`) → verify it's actually right → store in Postgres behind an adapter → serve the six primitives through one MCP gateway (freshness-gated caches) → a background gardener keeps it true and fills gaps. Downstream asks the map to *route*; truth is read *live*.

## 3. Not-Included (scope boundary)
No carving into owned/named/stable units (1.3). No unit experts (1.4). No answer generation or sandbox (05). No graph-explorer UI (06). Never stores truth; never asserts from map/cache.

## 4. Architecture & stages (direction-pointers on the *how*; precise on contracts in §5)
```
1.1 artifacts+perms → EXTRACT(code‖knowledge) → UNIFY+RESOLVE(entity resolution, weave perms, hierarchy)
  → COMPLETE(mechanical→CPG(agentic)→gap-fill→unknown) → VERIFY(ground+judge+exam)
  → STORE(Postgres+pgvector, behind adapter: system graph + lazy unit subgraphs)
  → SERVE via one MCP GATEWAY (six primitives; freshness-gated caches) → 02–06
  → MAINTAIN(gardener + learning router)
```
**Three layers:** Query (graph, gateway, semantic-cached) · Memory (cited per-unit briefs, prompt-cached — the agent reads these) · Truth (live reads, never cached). **Three-tier hierarchy:** resident system graph → lazy unit subgraphs → live truth (largest object held = system graph + the meeting's units ⇒ size-proof).

**Adopt (map-making):** codebase-memory-mcp (code structure), Cognee (knowledge graph + entity resolution + memory), Leiden (hierarchy), Joern/codebadger (deep completeness, agentic), Postgres+pgvector (store, behind adapter), one MCP gateway, semantic cache. **Build (the differentiated glue):** estate unification, entity-resolution tuning, completeness recovery, verification, the six primitives, capability routing, the query router, brief generation, the gardener.

**Escalation switches (OFF):** `DEEP_TRAVERSAL` (native graph engine behind adapter) · `KNOWLEDGE_GRAPH` (ship code-only, add Cognee later) · `CPG_COMPLETENESS` (Joern) · `ENTERPRISE_ACL` (inherited from 1.1).

## 5. Data contracts
Uses `Node`, `Edge`, `Unknown` from `AGENTS.md §Shared Types`. The **six primitives** are the MCP contract: `resolve_entity · relate · explain · verify · operate · coverage`. Every edge carries a `confidence` (`EXTRACTED`|`INFERRED`) and a required `citation` (ungrounded edges are dropped). Every node carries `freshness`, `acl_ref`, an optional `check_pointer` (address of truth) and `capability_refs` (for `operate`).

## 6. Acceptance criteria (the promises — verified on real data, never "exists")
Every criterion is an **outcome on a graph built from a real estate, with a threshold** — not "node X exists" or "edge Y is present." Both tiers must pass on the diverse real estates in `/fixtures/1.2/`.

**Tier-1 — technical (deterministic graders on a graph built from a real repo)**
- T1. **0** persisted edges lack a citation (grader: scan edges).
- T2. Measured edge validity ≥ the floor on a 100-edge human-labeled sample, well above raw ~43% (grader: grounding + calibrated judge).
- T3. **0** `verify`/`operate` results served from cache; a `verify` on a source-changed node returns the **new** live value (grader: stale-vs-live diff).
- T4. **0** cross-tenant reads succeed (grader: access matrix).
- T5. A source change marks **only** touched nodes stale (no full rebuild) (grader: delta check).
- T6. Every read is permission-filtered by `acl_ref` at the gateway (grader: unauthorized-principal query → empty).
- T7. The store is swappable behind the adapter with **0** downstream changes; the graph rebuilds by re-ingesting 1.1 artifacts (grader: swap + rebuild).

**Tier-2 — product (golden-dataset eval + threshold; on messy/single/big/multi-service/docs-heavy estates)**
- P1 · Unified map. A single query returns code + doc + ticket nodes for one unit, and the same entity across sources is **one** node (**≥90%** correct merges vs golden). *Grader: merge check + LLM-judge.*
- P2 · Relational answers. On **20 real "what breaks if X changes?"** questions with senior-engineer golden answers, **≥85%** return the expert's dependency set, cited, in **<1s**. *Grader: golden-set match + latency.*
- P3 · Resolve meaning. On **20 real ambiguous references** from transcripts, **≥90%** resolve to the correct node unspelled. *Grader: golden-node match.*
- P4 · Honest gaps. On questions about deliberately-unmapped areas, **100%** abstain + name a likely owner, **0%** bluff. *Grader: automated + LLM bluff-judge.*
- P5 · Live truth. On a claim whose source changed after ingest, `verify` returns the **new** value from a live read (never the stale map/cache), **100%**. *Grader: stale-vs-live.*
- P6 · Do-anything. On **5 task classes never designed for**, each is serviced by composing the six primitives with **0** graph-code changes; an uncovered demand falls back to exploration, never blocks. *Grader: code-diff = 0 + fallback trace.*
- P7 · Any repo. The identical pipeline builds a **good** graph on a messy repo, a single small repo, and a large monorepo — Tier-1 + P1–P6 all clear on each, **0** size/industry branches. *Grader: run the full suite per estate.*

**Done** = all Tier-1 pass + all Tier-2 clear threshold on every estate + adversarial suite clean (§7).

## 7. Testing strategy (eval-driven; real data, strictly graded)
**Golden dataset (`/fixtures/1.2/`).** Reuse 1.1's ingested estates + add a multi-service/cross-language repo and a docs-heavy repo, and at least one deliberately messy estate (dynamic dispatch, config routing) so the coverage ledger is non-empty. Build the full graph over each. Author **~100 human-reviewed goldens**: blast-radius Q&A, ambiguous-reference targets, edge-validity labels, unmapped-area expectations, live-truth cases, novel-task classes. Living asset.

**Graders (hybrid).** Deterministic/code graders for Tier-1 and the measurable Tier-2 parts (merge checks, latency, code-diff, stale-vs-live, permission matrix). **LLM-as-judge calibrated against the human gold set before it may gate** for blast-radius correctness, bluff detection, and validity. Metrics: correctness, groundedness, completeness, honesty, latency, cost. Thresholds set before building (above) are the bar.

**The build gate.** Done means every Tier-1 property holds and every Tier-2 eval clears threshold on **every** estate, and the **adversarial suite** finds no invariant violation:
- stale-cache attack (verify never returns cached/map value),
- cross-tenant probe, permission bypass + mid-session revoke,
- tool-poisoning/injection (gateway scans; injected instruction never acted on),
- hallucinated-edge induction (verifier drops/flags; validity stays ≥ floor),
- silent-gap probe (config-routed edge → recovered-with-citation or owned `unknown`, never absent),
- fabricated-citation sampling (every citation resolves to supporting content),
- gate probe (uncovered demand → graceful fallback, never hard "not found").

**The loop.** Build task → graders + adversarial on real estates → any miss or invariant break → **fresh context** + failure report → fix root cause → rebuild → repeat until all clear at threshold. Never weaken a criterion to pass. Stop only when the promises in §2 are proven on real data and the system resists attack.

## Gotchas (known wrong turns)
- Never assert a fact from the map or a brief — only from a live read at verify time. Never cache verify/operate.
- Entity-resolution string-match fails on abbreviations/synonyms — use hybrid (name + description embedding + structural proximity), not string equality.
- Raw LLM extraction is ~43% valid — every INFERRED edge MUST pass the validity judge or be dropped; EXTRACTED (parser) edges are trusted.
- The graph accelerates, never gates: an uncovered demand falls back to full agentic exploration, never returns "not found."
- A config-routed/dynamic edge is recovered-with-citation OR a typed `unknown` — never silently absent.
- Don't build a graph engine or a bi-temporal store — adopt (codebase-memory, Cognee, Postgres) behind the adapter.
