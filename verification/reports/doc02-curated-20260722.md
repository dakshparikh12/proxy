## Defects found & fixed

### DEFECT-02 (robustness) — webhook parser crashes on malformed payload
**Found by:** Layer 1 `test_meeting_metadata_never_crashes`, confirmed empirically:

| input | old result |
|-------|-----------|
| `{"data": None}` | `AttributeError: 'NoneType' has no attribute 'get'` |
| `{"data": 123}` | `AttributeError: 'int' has no attribute 'get'` |
| `{"data": {"participants": ["notadict"]}}` | `AttributeError: 'str' has no attribute 'get'` |

**Root cause:** `meeting_metadata` assumed `data` was a dict and each participant a
dict; `is_bot_removed` shared the same unguarded `payload.get("data", {}).get(...)`
pattern (its default `{}` is skipped when the key exists with a `None`/scalar value).
A corrupt/partial Recall delivery would raise straight out of ingest — the untested
AC-FAIL webhook-robustness area the sealed verdict flagged.

**Fix (real code, `services/transport/src/transport/events.py`):**
- Added `_as_dict()` coercion; `is_bot_removed`, `is_meeting_end` and `meeting_metadata`
  now degrade a non-object `data`/`participants`/entry to empty fields and coerce names
  to `str`, instead of raising. Well-formed classification is unchanged.

**Evidence of fix:** Layer 1 goes 1-failed→**6-passed** (`thorough`, 1000 examples);
ruff + `mypy --strict` clean; **28** sealed doc02 event/meeting/webhook tests still
pass. Commit `8862027`.
